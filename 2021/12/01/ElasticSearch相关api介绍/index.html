索引 Index

ES会索引所有字段，经过处理后写入一个倒排索引。查找数据的时候，直接查找该索引。所以，ES数据管理的顶层单位就叫做 Index（索引）。它是单个数据库的同义词。每个 Index （即数据库）的名字必须是小写。

创建文档

创建成功以后会得到下面的数据，__id为该记录id，如果没指定的话 es 会帮我生成这种随机id，result为我们执行的操作，_index为所属索引

注1：如果创建时报 [FORBIDDEN/12/index read-only / allow delete (api)];这个错可以尝试使用更新索引内容的方法，修改索引read_only_allow_delete属性为false.

注2：Document 可以分组，比如index_test这个 Index 里面，可以按类型分组（比如分为动物或者植物等），这种分组就叫做 Type，它是虚拟的逻辑分组，用来过滤 Document。

在 es7.x 之后取消了 type，均使用_doc同一文档类型。向指定的 /Index/Type 发送 PUT 请求，就可以在 Index 里面新增一条记录。比如，向/idx1/_doc发送请求，就可以新增一条人员记录

更新文档

  • 局部替换（存在就修改，不存在就追加）

  • 全量替换（直接全部覆盖文档内容）

每次更新以后，记录的 Id 没变，但是版本_version会＋1，操作类型（result）从created变成updated。

查询文档

返回结果的 took字段表示该操作的耗时（单位为毫秒），timed_out字段表示是否超时，hits字段表示命中的记录，里面子字段的含义如下。

  • total：返回记录数，本例是2条。
  • max_score：最高的匹配程度。
  • hits：返回的记录组成的数组。

返回的记录中，每条记录都有一个_score字段，表示匹配的程序，默认是按照这个字段降序排列。
